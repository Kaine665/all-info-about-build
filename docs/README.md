# `docs/` 目录说明

本目录**仅存放与 all-info-about-build 这个仓库自身相关的文档**（如何维护收件箱、CI、协作约定等），**不是**用来装「仓库里收集到的通用知识正文」的。

| 放置位置 | 用途 |
|----------|------|
| **`inbox/`** | 未整理的构建/分发/商业等笔记草稿（机器校验 frontmatter）。 |
| **根目录 `README.md` 等** | 仓库入口说明、流程约定。 |
| **`docs/`（本目录）** | **仅**本仓库的元文档；若暂无此类文件，目录可空或仅保留本说明。 |

与产品、网页、独立项目相关的长清单、方法论等，应放在 **`inbox/`**（草稿）或团队约定的**外部知识库**，不要默认塞进本仓库的 `docs/`。

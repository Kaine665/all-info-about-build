## 上下文

本条整理 **Bonjour!**（微信小程序 + 网页 `bonjour.bio` + NFC 实体触点）的公开信息与常见商业化路径，供本仓库内做产品、增长与收款方案对照时使用。信息来自 **官网/用户协议、创始人对外专访、行业报道与社区二手叙述** 的拼合；**用户数、融资额、收费明细以官方与小程序实时展示为准**。同名主体（如海外 `Bonjour.io`、香港零售「卓悦 Bonjour」等）与本文所述产品 **无必然关联**，检索时需甄别。

## 正文

### 1. 产品是什么（定位与形态）

- **定位（对外常见叙述）**：面向 **00 后极客、创业者、设计师、开发者** 等人群的 **AI 原生社区 + 数字名片网络**；从「可被看见的 Profile」切入，强调 **线下连接效率** 与 **合作/任务** 导向，而非纯内容信息流社区。
- **产品形态（「三位一体」常见说法）**：
  - **微信小程序**：主入口之一，便于在微信内分享与打开。
  - **网页主页**：`bonjour.bio` 等，便于跨端展示与传播。
  - **NFC 实体触点**：手机贴近卡片/贴纸等 → 唤起小程序或打开主页，降低「加好友/交换信息」摩擦。
- **核心功能（按公开描述归纳）**：
  - **NFC 碰一碰名片**：硬件 + 软件闭环。
  - **AI 匹配/助手**：报道中有 **Bonnie** 等角色，用于推荐「值得聊的人」或辅助连接（具体能力以产品内为准）。
  - **个人主页**：技能、项目、作品、兴趣等结构化展示。
  - **社区能力**：找合伙人、coffee chat、项目分享等（迭代节奏以官方版本为准）。
- **运营主体（以用户协议为准）**：《Bonjour 用户协议》（`bonjour.bio/terms`，曾标注更新于 2025-01-17）载明服务由 **杭州开始链接科技有限公司** 及其关联方提供；部分报道亦出现 **杭州起联科技** 等名称，**对外材料应以协议与工商登记为准**。

### 2. 发展脉络（公开报道口径，非内部时间轴）

- 团队自述路径常见版本：**先做偏创业者社区的方向 → 收敛到「数字名片 / Profile」切口 → 绑定高密度线下场景**（如黑客松、展会类活动）做冷启动与品牌传播。
- **融资**：多篇公开稿提及 **奇绩创坛（MiraclePlus）** 等投资方；**轮次、金额、时间** 在不同转载中存在差异，**以天眼查/官方披露为准**。
- **团队规模**：不同时间点口述从 **极小核心团队** 到专访中出现的 **约五人核心 + 兼职** 不等，仅作规模感参考。

### 3. 用户与人群（注意口径）

- 公开二手材料中出现过 **数万至十余万级** 等不同用户量级表述；**是否包含注册未活跃、是否含网页访客、统计截止时间** 往往未写清。
- **结论**：若用于对外 BP 或报道，务必向团队索取 **同一口径**（注册 / MAU / 付费客户等）与 **截止时间**。

### 4. 商业盈利模式（分层：已见证据 vs 常见路径）

#### 4.1 协议与官网能直接支撑的说法

- **用户协议**写明：Bonjour 服务 **可能包含需付费或附带其他限制条件的项目**；服务可用性、**价格**、内容与功能 **可能随版本与运营调整**（即存在 **官方收费/增值** 空间，但协议正文 **未列价目表**）。
- **官网首页**当前以登录与合规信息为主，**未见完整公开价目页**；实体卡等 **具体售价** 多在 **微信小程序内** 展示，需以客户端为准。

#### 4.2 创始人专访中的明确口述（任务/Agent 方向）

- 2025 年前后多篇转载的 **36 氪出海 / 腾讯新闻** 类专访中，Vincent 将平台方向描述为 **「Agent 时代的 Upwork」** 一类 **任务撮合网络**，并提到对 **服务/任务成交结果** 抽取 **约 15%** 的平台费用（口述比例，**以实际下单页与最新服务条款为准**）。

#### 4.3 与「社区 + 硬件 + B 端」常见模型对齐的备选收入（产品若演进到位）

下列为 **同类产品常见、且与 Bonjour 叙事相容** 的盈利模块，**是否已上线、占比多少** 需以 **小程序内购买项与合同** 核实：

1. **NFC 与活动物料**：实体名片卡、胸牌、贴纸、展会互动装置等 — **硬件毛利 + 批量定制**。
2. **企业 / 活动服务（B 端）**：黑客松、展会、沙龙的 **现场社交方案**（墙贴、打印、统一视觉与账号体系）、**企业数字名片与员工档案**。
3. **会员与增值服务（C 端）**：高级模板、主页自定义、**谁看过我** 类数据洞察、匹配或对接权益等。
4. **撮合抽成（中长期）**：项目外包、合作、招聘/人才介绍等 **交易佣金**（与专访口述方向一致）。
5. **品牌赞助与定向曝光**：创投、云厂商、开发者工具品牌等 **活动与社区赞助**。
6. **匿名化行业洞察 / API / 白标（远期）**：对 **合规与授权** 要求极高，通常只在 **B 端成熟、法务完备** 后开展。

### 5. 与泛社区产品的差异（战略备忘）

- **起点**：不仅是内容消费，而是 **身份（Profile）+ 线下触点（NFC）+ 连接动机（合作/任务）**。
- **闭环**：高密度线下场景带来 **第一批高信任用户**，硬件强化记忆点；若撮合层跑通，**LTV** 更依赖 **交易抽成 + B 端合同**，而非单靠广告。
- **风险点**：抽成模型依赖 **供给质量、履约、争议处理与合规**；用户规模叙事需 **口径统一**，避免与同名公司数据混淆。

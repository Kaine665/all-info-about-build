## 上下文

做运营与增长前，需要先对齐「我们是谁、推什么、为何推、推给谁、如何触达、触达后期望行为」。下列问题按本仓库 `inbox` 条目惯例整理为可逐条回答的清单，便于评审、吸收进正式文档（如 `README.md`、`docs/` 或产品 SPEC）前在此落稿。

## 正文

> 说明：下列每项下用简短段落或列表填写即可；若某问暂不适用，写「不适用」并简述原因。

### 1. 你是谁

- [ ] **主体身份**：个人 / 团队 / 公司 / 开源社区 / 其它（请写明）。
- [ ] **一句话定位**：用一句话说清「你提供什么价值、与同类差异」。
- [ ] **可信度依据**：经验、案例、数据、背书、许可证或公开作品等（可附链接占位，勿写密钥）。

### 2. 你想推广什么东西

- [ ] **推广对象**：具体产品、服务、内容、仓库、课程、活动等（名称 + 形态）。
- [ ] **核心交付物**：用户「得到」的是什么（功能、信息、结果、社群权益等）。
- [ ] **当前阶段**：内测 / 公测 / 正式 / 维护；与推广话术一致的版本或范围。

### 3. 你推广的目的是什么

- [ ] **主目标**（选填或补充）：认知 / 注册 / 试用 / 付费 / 留存 / 推荐 / 合作 / 其它。
- [ ] **可量化指标**：例如访问量、转化率、试用数、收入、Star 数等（写清时间窗口与口径）。
- [ ] **非目标（边界）**：明确不追求什么，避免动作发散。

### 4. 你的目标用户人群是哪些

- [ ] **人口与场景**：角色、行业、地区、设备/平台、使用场景。
- [ ] **痛点与动机**：他们为何需要你的推广对象。
- [ ] **非用户**：明确排除哪些人（同样有助于渠道与文案选择）。

### 5. 你怎么找到你的目标用户人群

- [ ] **渠道假设**：搜索引擎、社区、社交媒体、邮件、线下、合作伙伴、应用商店、开源生态等。
- [ ] **内容与触达方式**：教程、对比、案例、短视频、直播、文档、演讲等（与渠道对应）。
- [ ] **验证计划**：如何小成本验证渠道是否有效（A/B、落地页、小范围投放、访谈数量等）。

### 6. 目标用户找到你后，你希望他们怎么样

- [ ] **首选行动（CTA）**：例如注册、安装、订阅、加群、留资、贡献 PR、购买。
- [ ] **次要路径**：若首选未完成，备选动作是什么。
- [ ] **成功体验定义**：用户完成哪些步骤算「一次成功的接触」（与第 3 节指标可对齐）。

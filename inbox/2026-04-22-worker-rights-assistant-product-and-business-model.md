## 执行摘要

**工人维权助手**定位为：面向劳动者的**工作事实与争点记录工具**，帮助用户持续、结构化地记录与导出与用工相关的事实材料；**不提供法律服务、不提供个案法律意见、不代理任何争议解决程序**。

产品内核采用**争点导向**的可扩展架构：以劳动纠纷中常见争点为纲，映射为「应记录的事实要件」与字段/模板；新争点可随认知与反馈迭代加入。

商业模式采用组合策略：**免费核心 + 订阅增值**为主干，**按次或按包的材料整理与导出能力**为第二付费点，**机构与企业服务（ToB）**为规模化选项，**与律师/调解等合作分润**在合规成熟后作为补充。本文档供内部对齐与迭代使用，合规细节以正式法务审阅与监管要求为准。

# 第一部分：产品方案

## 1. 产品定位

| 维度 | 说明 |
|------|------|
| **产品名称（主品牌）** | 工人维权助手（对外可配合副标题强调「记录与整理工具」） |
| **品类** | 工具类 / 效率类；本质是**个人信息与工作事实的管理工具** |
| **边界** | 仅提供：记录、统计、时间线、清单、导出、备份等**工具能力**；不提供法律咨询、不代写文书、不承诺案件结果 |

## 2. 目标用户与场景

**主要用户**：与用人单位存在或可能发生用工争议风险的劳动者，包括但不限于制造业、建筑业、服务业、新业态用工等场景。

**核心场景**：

- **日常**：持续记录出勤、工时、工资到账、岗位变化等，形成习惯化流水。
- **争议前兆**：发现拖欠、克扣、违法解除苗头时，补齐历史区间记录与证据线索。
- **准备行动**：整理时间线、导出材料包，便于用户自行寻求法援、律师或行政救济（工具外完成）。

## 3. 核心价值主张

1. **可核对**：记录以日期、金额、时长、对象为主，便于与聊天记录、银行流水等第三方材料对照。
2. **争点对齐**：字段与模板围绕**常见仲裁争点**设计，减少「记了很多却用不上」的无效成本。
3. **可扩展**：新争点以「模块」形式加入，不推翻既有结构。
4. **边界清晰**：用户明确知晓产品**不是**律师事务所或维权代理。

## 4. 功能架构（争点导向）

以下为**逻辑模块**，具体 MVP 可裁剪实现顺序。

### 4.1 基础层（全用户）

- 用户与用工关系信息（单位名称、岗位、大致入职时间等，字段可渐进完善）。
- 通用「日记式」工作事实记录（文本 + 时间戳）。
- 本地存储与基础隐私设置。

### 4.2 争点模块（可迭代扩展）

每个模块包含：**说明（适用于何种情况）+ 引导字段 + 可选模板**。

| 争点方向 | 典型记录要点（与前期讨论一致） |
|----------|-------------------------------|
| 劳动关系 | 用人单位、工作地点、管理关系、合同与签收线索等 |
| 工资与拖欠 | 约定与结构、应发实发、发放日、扣款与理由摘要 |
| 加班与休息 | 延长工时、休息日、法定节假日上班的分段记录 |
| 合同订立与变更 | 签订时间、变更与续签线索 |
| 解除与终止 | 解除方式、日期、单位理由摘要、沟通线索 |
| 社保等（视产品策略） | 参保线索、与主张相关的侧面事实 |
| 工伤与安全事故 | 时间地点经过、就医与申报线索（客观描述为主） |
| 其他严重不公行为 | 时间地点、在场人、客观行为描述（避免煽动性表述） |

**扩展机制**：新增争点 = 新增模块说明 + 要件列表 + 字段/模板，在帮助中心与版本说明中同步告知用户。

### 4.3 增强能力（偏付费）

- 云同步与多设备、长期备份策略。
- 高级导出：PDF、表格、按争点汇总的时间线视图。
- 「材料整理」视图：按用户勾选争点生成结构化导出包（**仍为事实汇总，不含法律结论**）。

## 5. 关键流程（示意）

1. ** onboarding**：简要说明产品边界（非法律服务）；可选引导选择「当前最关心的争点类型」以预加载模块。
2. **日常记录**：默认低门槛流水；从争点模块进入时字段更结构化。
3. **回顾与导出**：按时间或按争点筛选 → 预览 → 导出至本地或用户指定存储。
4. **帮助与紧急提示**：提供官方救济渠道的一般性索引（如 12333 等公共信息，以当地公开信息为准，需定期核对）；**不替代**个案咨询。

## 6. 合规与产品表述

- **必须在显著位置重复展示**：本产品为信息记录与整理工具，**不构成法律意见**；个案策略请咨询执业律师或法律援助机构。
- **功能与文案**：避免胜诉承诺、避免替代专业判断的表述；材料包定位为「事实与时间的结构化汇总」。
- **数据与安全**：隐私政策清晰说明存储位置、加密与导出规则；敏感信息最小化展示。
- **各分发渠道**：主品牌可保持「工人维权助手」，若某渠道对标题敏感，可采用**同一应用、中性副标题或商店展示名**的变体，功能与描述保持一致，避免实质性误导。

## 7. 成功指标（建议）

| 阶段 | 指标示例 |
|------|----------|
| 验证期 | 激活后 7 日内再次打开率、人均记录条数、争点模块使用率 |
| 增长期 | 自然分享、商店评分、导出次数 |
| 商业期 | 付费转化率、续订率、材料包 ARPU、ToB 合同数 |

## 8. 风险与对策（产品侧）

| 风险 | 对策方向 |
|------|----------|
| 用户误以为可获得法律服务 | 强边界文案 + 帮助中心 FAQ + 流程内提示 |
| 审核对「维权」表述敏感 | 准备中性副标题与截图话术；功能与描述一致 |
| 情绪极端场景下的舆情 | 强调客观记录与合法救济入口，避免工具内煽动性运营 |
| 证据效力误解 | 说明工具生成内容为**用户自述整理**，效力结合原始证据由司法机关认定 |

## 9. 路线图建议

- **MVP**：基础记录 + 1～2 个核心争点模块（如劳动关系 + 工资 + 加班）+ 基础导出。
- **V1.1**：云同步、订阅体系、高级导出。
- **V1.2**：材料包按次付费、更多争点模块。
- **V2**：机构版试点、合作方对接（在合规评估后）。

# 第二部分：商业模式报告

## 1. 收入结构总览

采用**组合收入**，降低对单一渠道的依赖：

1. **订阅（主要经常性收入）**
2. **单次或按包的功能付费（高意愿场景）**
3. **ToB / 机构服务（项目型与年费）**
4. **合作分润（后期，强合规前提）**

可选补充：赞助、内容付费等，不作为早期支柱。

## 2. 订阅制

**对象**：长期、高频记录用户。

**典型权益**：

- 云同步与多设备；
- 高级导出格式与批量导出；
- 争点模板全量或增量解锁；
- 更长周期的备份与恢复策略。

**定价思路**：参照国内工具类应用常见区间，设置**月付 / 年付**（年付折扣）；具体数字需结合成本与用户调研标定。

**逻辑**：记工与维权准备是**中长期行为**，订阅与使用节奏匹配。

## 3. 单次 / 材料包付费

**对象**：争议临近、急需结构化输出的用户。

**典型交付**：按用户勾选争点生成时间线 + 清单 + 导出包（工具层）；可与订阅打包为「首月含一次材料包」等促销。

**逻辑**：**离行动越近，付费意愿越强**，适合作为 IAP 或站内一次性购买，与订阅形成互补而非替代。

## 4. ToB / 机构

**对象**：工会、法律援助中心、公益组织、行业培训或合规相关机构（视准入与资质）。

**形态**：批量账号、定制模板、脱敏统计、简易管理后台、培训与驻场支持（按需）。

**逻辑**：单合同金额高，弥补 C 端 ARPU 天花板；销售周期与交付成本较高，适合产品稳定后拓展。

## 5. 合作分润（后期）

**形态**：用户在明确知情与同意下，跳转至合作律师、调解或合规服务；按线索或成交分成。

**前提**：合作方资质审查、广告与推荐合规、用户协议与披露完整；**不宜作为冷启动主收入**。

## 6. 成本与毛利（定性）

主要成本项：云与存储、开发与维护、客服与合规咨询、支付手续费、应用商店分成、ToB 交付人力。

**优化方向**：存储分层、导出异步计算、分渠道评估获客成本。

## 7. 阶段与商业模式匹配

| 产品阶段 | 主推变现 |
|----------|----------|
| MVP～验证 | 免费为主，可选「早鸟订阅」或小范围年付 |
| 增长期 | 订阅正式化 + 单次材料包上线 |
| 成熟期 | ToB 试点；评估合作分润 |
